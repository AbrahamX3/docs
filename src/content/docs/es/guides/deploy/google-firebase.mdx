[Firebase Hosting](https://firebase.google.com/products/hosting) es un servicio provisto por la plataforma de desarrollo de Google [Firebase](https://firebase.google.com/), el cual puede ser usado para desplegar un proyecto de Astro.

Consulte nuestra guía separada para [desplegar servicios backend de Firebase](/es/guides/backend/google-firebase/), como bases de datos, autenticación y almacenamiento.

## Configuración del proyecto

Tu proyecto Astro se puede implementar en Firebase como un sitio estático o como un sitio renderizado del lado del servidor (SSR).

### Sitio estático

Tu proyecto Astro es un sitio estático por defecto. No necesitas ninguna configuración adicional para implementar un sitio Astro estático en Firebase.

### Adaptador para SSR

Para habilitar SSR en tu proyecto Astro e desplegar en Firebase, agrega el [adaptador Node.js](/es/guides/integrations-guide/node/).

:::nota
Para desplegar un sitio SSR Astro en Firebase se requiere el [plan Blaze](https://firebase.google.com/pricing) o superior.
:::

## Cómo desplegar

1. Instala [Firebase CLI] (https://github.com/firebase/firebase-tools). Esta es una herramienta de línea de comandos que le permite interactuar con Firebase desde la terminal.

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npm install firebase-tools
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm install firebase-tools
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn add firebase-tools
    ```
    </Fragment>
  </PackageManagerTabs>

2. Autentique Firebase CLI con tu cuenta de Google. Esto abrirá una ventana del navegador donde podrá iniciar sesión usando tu cuenta de Google.

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npx firebase login
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm exec firebase login
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn firebase login
    ```
    </Fragment>
  </PackageManagerTabs>

3. Habilite soporte experimental para frameworks web. Esta es una función experimental que permite que Firebase CLI detecte y configure los ajustes de implementación de Astro.

 <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npx firebase experiments:enable webframeworks
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm exec firebase experiments:enable webframeworks
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn firebase experiments:enable webframeworks
    ```
    </Fragment>
  </PackageManagerTabs>

4. Inicialice Firebase Hosting en su proyecto. Esto creará un archivo `firebase.json` y `.firebaserc` en la raíz de su proyecto.

  <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npx firebase init hosting
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm exec firebase init hosting
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn firebase init hosting
    ```
    </Fragment>
  </PackageManagerTabs>

5. Despliega tu sitio en Firebase Hosting. Esto construirá tu sitio Astro y lo implementará en Firebase.

   <PackageManagerTabs>
    <Fragment slot="npm">
    ```shell
    npx firebase deploy --only hosting
    ```
    </Fragment>
    <Fragment slot="pnpm">
    ```shell
    pnpm exec firebase deploy --only hosting
    ```
    </Fragment>
    <Fragment slot="yarn">
    ```shell
    yarn firebase deploy --only hosting
    ```
    </Fragment>
  </PackageManagerTabs>
